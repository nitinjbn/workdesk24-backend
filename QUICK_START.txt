🚀 SETUP (5 minutes)
─────────────────────────────────────────────────────────────────

1. Configure Environment
   cp .env.example .env
   # Edit .env - set DB_PASSWORD and SMTP credentials

2. Setup Database
   CREATE DATABASE workdesk24;
   npm run db:migrate

3. Start Server
   npm run dev
   # Server runs on http://localhost:3000

📧 EMAIL SETUP (Gmail)
─────────────────────────────────────────────────────────────────

1. Enable 2FA on Google Account
2. Generate App Password:
   https://myaccount.google.com/apppasswords
3. Update .env:
   SMTP_HOST=smtp.gmail.com
   SMTP_PORT=587
   SMTP_USER=your-email@gmail.com
   SMTP_PASSWORD=your-16-char-app-password
   ADMIN_EMAIL=admin@example.com

🔐 AUTHENTICATION FLOW
─────────────────────────────────────────────────────────────────

1. POST /api/auth/login with {email, password}
2. Receive JWT token in response
3. Include in requests: Authorization: Bearer <token>
4. Token valid for 7 days (configurable in .env)


💡 TROUBLESHOOTING
─────────────────────────────────────────────────────────────────

Issue: "Token is required"
→ Add Authorization header: Bearer <your-token>

Issue: Email not sending
→ Check SMTP credentials in .env
→ For Gmail: use App Password, not regular password

Issue: Database connection error
→ Verify MySQL is running
→ Check DB credentials in .env
→ Ensure database exists: CREATE DATABASE workdesk24;

Issue: Rate limit hit
→ Wait for window to reset (check Retry-After header)
→ Or adjust limits in src/config/rateLimit.js
